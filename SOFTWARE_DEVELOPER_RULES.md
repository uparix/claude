# 1. No Prototype

- Just make the application. Polish as you go. Don't depend on polish happening later. 
- Always maintain constantly shippable code.

# 2. Always run

- It's incredibly important that your application can always be run by your team. 
- Bulletproof your application by providing defaults upon load failure.

# 3. Keep your code simple

- Keep your code absolutely simple. 
- Keep looking at your functions and figure out how you can simplify further.

# 4. Tools

- Great tools help make great applications. 
- Spend as much time on tools as possible.

# 5. Testing

- We are our own best testing team and should never allow anyone else to experience bugs or see the application crash. 
- Don't waste other's time. 
- Test thoroughly before checking in your code.

# 6. Bugs

- As soon as you see a bug, you fix it. Do not continue on. 
- If you don't fix your bugs your new code will be built on a buggy codebase and ensure an unstable foundation.

# 7. Systems

- Use a superior development system than your target.- Use the best tools available money can buy.

# 8. Code

- Write your code for this application only - not for a future application. 
- You're going to be writing new code later because you'll be smarter.

# 9. Design

- Encapsulate functionality to ensure design consistency. This minimises mistakes and saves design time.
- Always make your application look nice. It’s how your application is received.

# 10. Communication

- Try to code transparently. Tell your lead and peers exactly how you are going to solve your current task and get feedback and advice. 
- Do not treat application programming like each coder is a black box. The project could go off the rails and cause delays.

# 11. Creative Art

- Programming is creative art form based in logic. 
- Every programmer is different and will code differently. 
- It's the output that matters.
